# H3C TV Control 自定义集成需求文档

## 1. 背景与目标

本项目通过 H3C S5550 交换机的 ACL 规则，控制多台 Sony 电视是否允许访问互联网。现有 YAML 方案（`command_line` + `template switch`）已能工作，但存在状态反馈延迟、乐观模式与滑块显示冲突、儿童管理难以扩展等问题。

自定义集成 `h3c_tv_control` 的目标：

- 替代现有 YAML 方案，提供原生 Switch 实体和标准滑块 UI
- 复用现有 Telnet/ACL 控制逻辑（`scripts/tv_internet/tv_internet_control.py`）
- 提供儿童上网时间管理（单次时长、每日总时长、允许时间段）
- 为后续科学上网出口、路由策略扩展打下基础

## 2. 用户故事

作为家长，我希望对每台电视（主卧/客厅/老人房/书房）能够：

1. 在 Home Assistant 中手动开/关电视网络
2. 设置「单次最多上网 X 分钟」
3. 设置「每天总共最多上网 Y 分钟」
4. 设置「仅在某时间段允许上网」（如 18:00–20:00）
5. 超出任一条件时自动断网
6. 每天 0 点自动重置当日统计

## 3. 功能需求（第一期）

| 功能 | 实体类型 | 说明 |
|------|----------|------|
| 电视上网开关 | Switch | 4 台电视，点击后立即更新 UI，后台轮询 H3C 校正 |
| 儿童控制开关 | Switch | 每台电视可单独启用/禁用儿童限制 |
| 单次允许分钟 | Number | 默认 30 分钟，范围 5–180 |
| 每日允许分钟 | Number | 默认 90 分钟，范围 10–480 |
| 允许开始时间 | Time | 默认 18:00 |
| 允许结束时间 | Time | 默认 20:00 |
| 今日已用分钟 | Sensor | 只读，每日 0 点重置 |
| 本次剩余分钟 | Sensor | 只读，会话进行中显示倒计时 |
| 自动断网 | 内部逻辑 | 会话超时 / 日总量超限 / 超出时间段 |
| 状态轮询 | Coordinator | 默认 60 秒查询交换机 ACL |

## 4. 非功能需求

- 交换机密码通过 Config Flow 配置，存入 `config_entry.data`，不写入代码
- Telnet 操作通过 `async_add_executor_job` 执行，不阻塞 HA 事件循环
- 交换机不可达时，相关实体标记 `unavailable`
- 配置变更后无需重启 Home Assistant
- 儿童限制运行时状态持久化到 `.storage`，HA 重启后可恢复

## 5. 边界与异常

| 场景 | 处理方式 |
|------|----------|
| 手动在交换机上改 ACL | 轮询后自动校正 HA 状态 |
| 儿童控制启用时家长手动开网 | 仍受时间段和日总量限制 |
| 跨天会话（23:50 开启，00:10 结束） | 按自然日分别统计已用时间 |
| 不在允许时间段内尝试开网 | 拒绝开启，记录 warning 日志 |
| 交换机 Telnet 失败 | 实体 unavailable，保留上次已知状态 |

ACL ID: 3000

## 7. 儿童限制规则优先级

1. 不在允许时间段 → 不允许开网；已开网则到点自动关
2. 今日总量已用完 → 不允许开网
3. 单次时长到 → 自动关网，今日总量继续累计
